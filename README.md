# Auth UI

A lightweight authentication interface built with **HTML, CSS, and JavaScript**.  
The project focuses on a clean login/register experience with smooth page transitions and an animated particle network background.

This is a frontend-only UI project meant for learning, experimentation, or integration into a larger application.

---

## Overview

This interface provides two authentication views:

- **Login page**
- **Register page**

Both are contained in a single card layout and transition smoothly between each other.  
The background uses a **canvas-based particle network animation** that reacts to cursor movement, giving the interface a modern feel.

---

## Features

- Sliding **Login / Register page transition**
- **Interactive particle network** background using HTML Canvas
- **Password visibility toggle**
- **Font Awesome input icons**
- Modern **glass-style card layout**
- Clean and minimal **UI design**
- Fully **frontend-based (no frameworks required)**

## Project Structure
auth-ui
│
├── index.html
├── style.css
├── script.js
└── README.md


**index.html**  
Contains the structure of the login and register pages.

**style.css**  
Handles layout, theme colors, animations, and UI styling.

**script.js**  
Controls page switching, password toggling, and the particle network animation.

## Running the Project

This project does not require a build system or package manager.

1. Download or clone the repository
2. Open the project folder
3. Open `index.html` in any modern browser

Alternatively, if you use **VS Code**, you can run it with the **Live Server extension**.

## Technologies Used

- HTML5
- CSS3
- Vanilla JavaScript
- HTML Canvas API
- Font Awesome

No frameworks or libraries were used so the code remains easy to understand and modify.

---

## Customization

You can easily modify the UI by editing the CSS file.

Examples:

- Change theme colors
- Adjust animation speed
- Modify particle density
- Update typography

The particle behavior can be adjusted inside `script.js`.

## Notes

This project focuses on **frontend interface design** only.  
It does not include backend authentication or database integration.

To use it in a real application, connect the form inputs to a backend service or API.
